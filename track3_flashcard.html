The RTCFC framework — structure every prompt around these five components
R
Role
Who Claude should be. "You are a senior BD strategist at a behavioral health company."
T
Task
The exact deliverable. "Write a 3-paragraph cold email to a health system CFO."
C
Context
What Claude needs to know. "The CFO is focused on total cost of care. Avoid clinical jargon."
F
Format
How it should look. "Three paragraphs. No bullet points. Under 200 words."
C
Constraints
What to avoid. "Do not mention specific pricing. Do not use first-person plural."
Specificity
Claude takes the shortest path. Make the destination specific.
A 500-word blog post for health system COOs
Something for our website
Health system CFOs focused on total cost of care
Healthcare people
3 bullets, each under 15 words, active voice
Make it shorter
Grounding with examples
Show, don't describe
Examples narrow the space of acceptable outputs. One good example beats three paragraphs of description.
Separate from instructions
Use labeled sections so Claude distinguishes examples from the task:
[EXAMPLES]
Value prop: BHIQ identifies high-risk members before avoidable utilization...
[END EXAMPLES]

Now write a value prop for: [new audience]
Multi-step prompting
Each step = one focused RTCFC prompt
Break complex tasks into sequential prompts. Each step builds on confirmed output from the last.
When steps help
More than two distinct phases. Output of one phase shapes the next. You need to review intermediate outputs.
Step 1: Summarize key terms → review → Step 2: Identify implications → review → Step 3: Draft recommendation
Meta-prompting
Use Claude to write prompts for Claude
Most valuable for recurring workflows. Three methods:
Prompt creation
"Write Project instructions for a BD outreach workflow that produces value prop emails for health system CFOs."
Prompt analysis
"Here are my current Project instructions. How does Claude interpret them? What is unclear or missing?"
Automated iteration
"Write 3 versions of this prompt optimized for: CFO, clinical director, Medicaid director."
Chain-of-thought + gating
Manual CoT (most reliable)
You ask each question, review, build the next on confirmed output. No assumptions compound.
Single-prompt CoT
"Before your recommendation, reason through contract terms, market access, and resources explicitly. Then conclude."
Gating structure
Enforces stops between stages. Three required elements:
1. Stopping rule: "Do not begin next stage until I reply GO."
2. Prescribed gate text Claude must output
3. Your activation keyword: GO
Tone, framing, and prompt debugging
Calm directives outperform shouted rules
Always use verified data. If unavailable, label estimates.
NEVER fabricate data!!!
Positive framing outperforms negation
Respond in prose paragraphs
Don't use bullet points
Only use real data
Don't use mock data
Prompt debugging checklist
□ Is the task clearly named?
□ Is the audience or reader specified?
□ Is the format or structure defined?
□ Are constraints written as positive requirements?
□ Would an example help narrow the output space?
□ Is the prompt asking for one thing or several?
Hallucination check
🟢 Use directly
Reasoning, drafts from your own material, formatting, brainstorming, rewriting
🟡 Spot-check
Named organizations, general legislation, broad industry statistics, named roles
Web search the claim. Confirm the source exists.
🔴 Always verify
Specific citations, named studies, statistics attributed to a source, DOIs, dates of publications
Ask Claude for the full citation + DOI. Confirm in PubMed or Google Scholar.
The more specific and attributable the claim → the higher the verification requirement
